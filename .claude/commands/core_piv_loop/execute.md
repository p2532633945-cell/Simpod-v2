# 执行：从计划实施

## 要执行的计划

阅读计划文件：`$ARGUMENTS`

## 执行指令

### 1. 阅读并理解

- **仔细阅读整个计划**
- 理解所有任务及其依赖关系
- 注意要运行的验证命令
- 审阅测试策略

### 2. 按顺序执行任务

对于"分步任务"中的**每个**任务：

#### a. 导航到任务
- 识别文件和所需操作
- 如修改则读取相关现有文件

#### b. 实施任务
- 严格按照详细规格执行
- 保持与现有代码模式一致
- 包含适当的类型提示和文档
- 在适当位置添加结构化日志

#### c. 随时验证
- 每次文件更改后，检查语法
- 确保导入正确
- 验证类型正确定义

### 3. 实施测试策略

完成实施任务后：

- 创建计划中指定的所有测试文件
- 实施所有提到的测试用例
- 遵循概述的测试方法
- 确保测试覆盖边缘情况

### 4. 运行验证命令

按计划顺序执行**所有**验证命令：

```bash
# 完全按计划中指定运行每个命令
```

如果任何命令失败：
- 修复问题
- 重新运行命令
- 仅在通过后继续

### 5. 最终验证

完成前：

- ✅ 计划中的所有任务已完成
- ✅ 所有测试已创建并通过
- ✅ 所有验证命令通过
- ✅ 代码遵循项目约定
- ✅ 已添加/更新文档（如需要）

- 创建的测试文件

### 准备提交
- 确认所有更改完成

## 注意事项

- 如果遇到计划中未解决的问题，记录它们
- 如果需要偏离计划，解释原因
- 如果测试失败，修复实施直到通过
- 不要跳过验证步骤
