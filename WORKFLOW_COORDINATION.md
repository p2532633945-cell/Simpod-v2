# 工作流协调：PROGRESS.md 与 Plan 文件

> 本文档解释 PROGRESS.md 和 Plan 文件之间的关系和协调机制

---

## 核心概念

```
PROGRESS.md              Plan 文件
    ↓                        ↓
高层任务跟踪          详细实施计划
    ↓                        ↓
待办事项（列表）        具体步骤（操作）
    ↓                        ↓
任务粒度：功能级          任务粒度：文件/代码级
```

---

## 两个层次的任务

### PROGRESS.md：项目级别的任务

**特点**：
- 高层、面向功能
- 描述"要做什么"
- 按优先级排序
- 跨多个对话/工具共享

**示例**：
```markdown
- [ ] 初始化 Next.js 项目（🔥 高优先级）
- [ ] 设置 Supabase 客户端（🔥 高优先级）
- [ ] 实现热区服务（⚡ 中优先级）
- [ ] 设计 UI 组件库（📋 低优先级）
```

### Plan 文件：功能级别的详细计划

**特点**：
- 低层、面向实施
- 描述"如何做"
- 包含具体文件、代码示例
- 单次使用（执行后归档）

**示例**：
```markdown
## 实施计划

### 任务 1：创建 Next.js 项目
- CREATE `package.json`
- CREATE `tsconfig.json`
- CREATE `app/layout.tsx`

### 任务 2：配置 Tailwind CSS
- UPDATE `tailwind.config.ts`
- ADD `@tailwindcss/` 指令到 `app/globals.css`
```

## Plan 文件中的 PROGRESS 引用格式

### 必需的章节

```markdown
## PROGRESS.md 关联

**实现任务**：初始化 Next.js 项目
**优先级**：🔥 高
**更新后操作**：完成后更新 PROGRESS.md，将此任务移至"已完成的工作"
```

### 完成后的更新章节

```markdown
## 完成后：更新 PROGRESS.md

执行完成后，必须更新 PROGRESS.md：

1. 将任务从"待办事项"移至"已完成的工作"
2. 添加完成日期和简要描述
3. 更新"最后更新"和"更新者"

**示例**：
```markdown
### 2026-03-07: 初始化 Next.js 项目
**状态**: ✅ 已完成
**更新者**: Claude Code
**描述**: 使用 npx create-next-app 创建项目基础结构
```
```

## 常见问题

### Q1：如果 PROGRESS.md 没有我想要做的任务怎么办？

**A**：
1. 如果是新功能/重要任务：先添加到 PROGRESS.md 的"待办事项"
2. 如果是小调整/临时任务：可以在 Plan 中标注为"临时规划任务"

### Q2：一个 PROGRESS 任务可以对应多个 Plan 吗？

**A**：可以。对于大型功能，可以拆分为多个 Plan：

```
PROGRESS 任务：实现热区功能

Plan 1：initialize-nextjs.md         - 基础架构
Plan 2：implement-hotzone-service.md  - 服务层
Plan 3：create-hotzone-ui.md         - UI 组件
```

每个 Plan 完成后，更新 PROGRESS.md 的任务进度。

### Q3：Plan 执行失败怎么办？

**A**：
1. Plan 中包含回滚步骤
2. 更新 PROGRESS.md 的"已知问题"部分
3. 标记任务为"进行中"状态
4. 记录失败原因和下一步行动

### Q4：多工具协作时，Plan 文件位置重要吗？

**A**：重要。Plan 文件统一放在 `.agents/plans/` 目录，所有工具都应该：

- 从这个目录读取 Plan
- 创建新 Plan 到这个目录
- 完成后不删除 Plan（可以归档或保留）

## 最佳实践

### 1. 保持粒度一致

- PROGRESS：功能级（用户可理解的）
- Plan：实现级（开发者可执行的）

### 2. 明确引用

- 每个 Plan 必须明确说明实现哪个 PROGRESS 任务
- Plan 完成后必须包含更新 PROGRESS 的指令

### 3. 及时更新

- Plan 创建后，PROGRESS 中的任务状态保持不变
- Plan 执行完成后，立即更新 PROGRESS
- 不要等到所有 Plan 都完成才更新

### 4. 优先级驱动

- 优先实现 PROGRESS 中高优先级（🔥）的任务
- 低优先级（📋）任务可以延后或合并
