## 🧠 战略背景

### 心流被打断的三个根本原因

```
1. 听到不懂 → MARK → 等待转录（感知延迟 > 2s → 心流断裂）
2. 想回去确认 → 手动拖进度条（操作成本高）
3. 复盘时找不到 → 搜索不准（关键词记不清楚）
```

### 两种泛听模式

**模式 A（泛听模式）**：专注听，MARK 只打时间戳，转录后台慢慢处理，事后复盘时展示完整文字。
- 对延迟不敏感
- 对截取范围精准度要求高（句子完整）
- 适用场景：无官方转录的播客

**模式 B（即时模式）**：MARK 后立刻看到转录文字，立即回溯确认。
- 要求 < 1s 感知响应
- 数据来源：官方转录 / 预转录缓存
- 适用场景：有官方转录的播客 / 已预转录的剧集

**系统自动感知，用户无需手动切换。**

## 🔬 转录数据战略

### 官方转录来源
- **Podcast Namespace 2.0** `<podcast:transcript>`：免费，~15% 播客覆盖
- **Apple Podcasts**：部分节目有官方字幕
- **Spotify**：部分节目提供转录（需要 API 接入）

### 第三方转录数据源（待探索）

| 服务 | 类型 | 成本 | 使用方式 | 建议时机 |
|------|------|------|---------|----------|
| **Taddy.org** | 播客索引 + 部分转录 | 免费 API（有限额）| 即时 API 调用 | 现在可以集成 |
| **Podscribe.ai** | 主流播客全文转录 | 付费订阅 | 批量下载存 DB | 用户量 > 1000 时 |
| **Listen Notes API** | 播客搜索 + 元数据 | 免费层 500次/月 | 补充剧集元数据 | 现在可以集成 |
| **AssemblyAI** | 转录服务（更高质量）| $0.37/小时 | 替代 Groq（高质量需求）| 付费用户专用 |
| **OpenAI Whisper** | 转录服务 | $0.006/min | 备选转录引擎 | 质量对比测试 |

### 第三方数据源详细说明

**Taddy.org**
- 免费 API，提供播客搜索、剧集元数据、部分转录片段
- 使用方式：按 podcast RSS URL 查询，获取已有转录
- 限制：覆盖率有限，非全文转录
- 建议：现在就可以集成，作为官方转录的补充来源

**Podscribe.ai**
- 专门做播客转录的服务，覆盖主流英文播客（Top 5000+）
- 支持批量下载，可存入自己数据库
- 成本：约 $99/月起（企业版）
- 建议：用户量达到 1000+ 后评估，转录复用后单用户成本极低

**Listen Notes**
- 最大的播客数据库，有 API
- 主要提供元数据（剧集信息、描述、时长）
- 部分高级端点提供转录
- 免费层够用于元数据补充
- 建议：现在集成，用于播客信息完善

### 自建转录复用策略

```
转录触发逻辑：
  用户 MARK
    → 查  transcripts 表（audio_id + 时间范围）`n    → 命中 → 直接返回（0 API 调用）`n    → 未命中 → Groq 转录 → 存入 transcripts 表`n    → 下次任何用户 MARK 同一段 → 复用`n`n预转录策略：`n  剧集 duration ≤ 30min`n    → 用户首次播放时后台触发全量转录`n    → 成本 $0.056/集（¥0.40）`n    → 存入 transcripts 表，永久复用`n`n官方转录策略：`n  剧集列表加载`n    → 检测 podcast:transcript RSS 标签`n    → 后台静默下载 VTT/SRT`n    → 解析存入 transcripts 表`n    → MARK 时零 API 调用，零延迟`n```
