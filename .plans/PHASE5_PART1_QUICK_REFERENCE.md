# Phase 5 第一阶段 - 快速参考

## 🎯 5 个核心功能

### 1️⃣ 热区编辑（P5-4）- 1.5 day
**用户故事**：作为用户，我想编辑已创建的热区

**关键功能**：
- 编辑时间范围（start_time, end_time）
- 编辑标题和描述
- 删除热区

**UI 位置**：复盘页面 (`/hotzones`) 的热区列表项

**API**：
```
PATCH /api/hotzones/:id
DELETE /api/hotzones/:id
```

---

### 2️⃣ 热区回溯（P5-5）- 2 day
**用户故事**：作为用户，我想快速回溯音频

**关键功能**：
- 快退 5s/10s/30s
- 快进 5s/10s/30s
- 进度条拖拽
- 键盘快捷键（← →）

**UI 位置**：播放器下方 (`PlaybackControls`)

**API**：无（纯前端）

---

### 3️⃣ 热区统计（P5-6）- 1 day
**用户故事**：作为用户，我想查看学习进度

**关键功能**：
- 热区数量统计
- 学习进度追踪
- 时间分布分析（图表）

**UI 位置**：新增统计页面或复盘页面顶部

**API**：
```
GET /api/analytics
```

---

### 4️⃣ 转录编辑（P5-7）- 1.5 day
**用户故事**：作为用户，我想修正转录错误

**关键功能**：
- 编辑转录文本
- 撤销/重做
- 添加注释

**UI 位置**：转录面板 (`TranscriptStream`)

**API**：
```
PUT /api/transcripts/:id
```

---

### 5️⃣ 转录搜索（P5-8）- 1 day
**用户故事**：作为用户，我想搜索转录内容

**关键功能**：
- 全文搜索
- 高亮匹配词
- 按关键词过滤热区

**UI 位置**：复盘页面搜索框

**API**：
```
GET /api/transcripts/search?q=keyword
```

## 🛠️ 技术要点

### 后端实现
- 所有 API 都需要 user_id 过滤（用户隔离）
- 所有修改操作都需要权限检查
- 添加适当的数据库索引

### 前端实现
- 使用 Zustand 管理状态
- 使用 React Query 缓存数据
- 实现乐观更新（optimistic update）

### 数据库
- 添加全文搜索索引（转录搜索）
- 添加统计视图（热区统计）
- 考虑分区（大数据量）

## 📞 需要帮助？

### 常见问题

**Q: 如何实现撤销/重做？**
A: 使用 `immer` 库或手动维护历史栈

**Q: 如何优化搜索性能？**
A: 添加全文搜索索引，使用分页，限制结果数量

**Q: 如何处理编辑冲突？**
A: 使用版本控制或乐观锁机制

**Q: 如何测试用户隔离？**
A: 创建两个测试账户，验证数据不可见
