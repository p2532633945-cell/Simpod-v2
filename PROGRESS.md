## 🔴 PWA 安装横幅问题 — 根本性解决方案

### 问题诊断
- ✅ manifest.json 已修复（401 → 200）
- ✅ maskable icons 已生成
- ❌ 但 Android Chrome 仍不显示安装横幅

### 根本原因分析
Chrome PWA 安装横幅需要满足 **Web App Install Criteria**：
1. ✅ manifest.json 有效
2. ✅ Service Worker 已激活
3. ✅ HTTPS 访问
4. ❌ **用户参与度** — Chrome 要求用户在网站停留 30s+ 或交互 2+ 次才显示横幅
5. ❌ **manifest 缓存** — 旧 manifest 被浏览器缓存，即使服务器更新也不会重新拉取

### 根本性解决方案

**方案 1：强制 manifest 刷新（推荐）**
```html
<!-- layout.tsx 中添加 -->
<link rel="manifest" href="/manifest.json?v=3" />
```
- 每次部署时递增 `v` 参数
- 强制浏览器重新拉取 manifest
- 同时在 manifest.json 中添加 `version` 字段

**方案 2：Service Worker 主动更新 manifest**
```javascript
// sw.js 中添加
self.addEventListener('message', (event) => {
  if (event.data && event.data.type === 'SKIP_WAITING') {
    self.skipWaiting()
  }
})
```
- 在 manifest 更新时通知 SW 立即激活
- 清除旧缓存

**方案 3：用户引导 + 手动安装提示**
```typescript
// 检测 beforeinstallprompt 事件
window.addEventListener('beforeinstallprompt', (e) => {
  // 保存事件，在用户交互后触发
  deferredPrompt = e
  // 显示自定义安装按钮
  showInstallButton()
})
```
- 不依赖 Chrome 自动横幅
- 在关键位置（如首页 hero、播放器底部）显示自定义「安装应用」按钮
- 用户点击时调用 `deferredPrompt.prompt()`

### 执行步骤
1. 实施方案 1（manifest 版本控制）
2. 实施方案 3（自定义安装提示）
3. 测试验证
