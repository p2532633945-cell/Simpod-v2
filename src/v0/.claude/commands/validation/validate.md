# 验证：完整验证流程

## 目标

执行完整的验证流程以确保代码质量和功能正确性。

## 验证级别

### 级别 1：语法与类型检查

```bash
# TypeScript 类型检查
npx tsc --noEmit

# ESLint（如配置）
npm run lint

# Prettier 格式检查
npm run format:check
```

**如果失败**：
- 修复类型错误
- 修复 lint 警告
- 运行格式化：`npm run format`

### 级别 2：单元测试

```bash
# 运行所有测试
npm test

# 运行特定测试文件
npm test -- hotzone.test.ts

# 测试覆盖率
npm test -- --coverage
```

**如果失败**：
- 检查测试输出
- 修复失败的测试
- 确保所有测试通过

### 级别 3：集成测试

```bash
# 运行集成测试
npm run test:integration

# 如有后端 API，测试端点
# 或使用 Postman/类似工具
```

**如果失败**：
- 检查 API 响应
- 验证数据库状态
- 检查网络连接

### 级别 4：构建验证

```bash
# 生产构建
npm run build

# 验证构建输出
ls -la dist/
```

**如果失败**：
- 检查构建错误
- 修复依赖问题
- 验证环境配置

### 级别 5：手动验证

根据功能进行手动测试：

#### 功能测试清单
- [ ] 创建新热区并保存
- [ ] 播放音频并标记锚点
- [ ] 搜索播客并选择 episode
- [ ] 查看转录文本
- [ ] 扩展现有热区

#### UI 测试
- [ ] 响应式设计（桌面/移动）
- [ ] 加载状态显示
- [ ] 错误状态处理
- [ ] 空状态显示

#### 性能检查
- [ ] 页面加载时间 < 3秒
- [ ] 交互响应 < 100ms
- [ ] 无明显内存泄漏

### 级别 6：代码审查检查点

- [ ] 代码符合 CLAUDE.md 约定
- [ ] 无硬编码值（使用环境变量）
- [ ] 错误处理完整
- [ ] 日志记录适当
- [ ] 类型定义完整
- [ ] 组件拆分合理
- [ ] 性能优化考虑

## 快速验证

仅运行基础检查：

```bash
npx tsc --noEmit && npm test && npm run build
```

## 注意事项

- **不要跳过验证**：每个级别都很重要
- **立即修复失败**：不要在损坏状态下继续
- **文档化问题**：记录任何发现的问题
- **保持测试绿色**：持续集成要求
