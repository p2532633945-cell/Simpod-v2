## 详细修复清单

### 修复 1: Supabase 错误处理 ✅

**文件**: `src/services/supabase.ts`

**改进点**:

1. **saveTranscript() 函数**
   - 原: 错误被吞掉，无法追踪
   - 新: 返回 `{ success: boolean; error?: string }`
   - 新: 添加指数退避重试（最多 2 次）
   - 新: 区分网络错误和业务错误
   - 新: 改进错误日志记录

2. **findExistingTranscript() 函数**
   - 原: 无法区分"无结果"和"查询失败"
   - 新: 添加重试机制（最多 1 次）
   - 新: 改进错误日志记录
   - 新: 更清晰的错误信息

**影响**:
- 转录缓存失败现在可以追踪
- 网络错误会自动重试
- 减少重复 API 调用

---

### 修复 2: API 代理超时和重试 ✅

**文件**: 
- `src/app/api/groq-proxy/route.ts`
- `src/app/api/rss-proxy/route.ts`

**改进点**:

1. **Groq 代理**
   - 原: 无超时设置，可能无限期挂起
   - 新: 30 秒超时
   - 新: 2 次指数退避重试
   - 新: 区分超时错误（504）和其他错误（500）

2. **RSS 代理**
   - 原: 无超时设置，可能无限期挂起
   - 新: 15 秒超时
   - 新: 2 次指数退避重试
   - 新: 区分超时错误（504）和其他错误（500）

**新增函数**:
- `fetchWithTimeout()` - 带超时的 fetch 包装
- `fetchWithRetry()` - 带重试的 fetch 包装

**影响**:
- 服务不再无限期挂起
- 网络波动时自动重试
- 更清晰的错误信息

### 修复 3: 播放器状态同步 ✅

**文件**: `src/stores/playerStore.ts`

**改进点**:

1. **新增 cleanup() 方法**
   - 重置所有播放器状态
   - 暂停音频并重置时间
   - 清空热区列表
   - 清空音频引用

2. **改进 setIsPlaying() 方法**
   - 原: 播放失败时没有错误处理
   - 新: 添加 `.catch()` 处理播放失败
   - 新: 播放失败时自动重置状态

**影响**:
- 快速切换播客时状态能正确清理
- 播放失败时不会导致状态混乱
- 更好的错误恢复
